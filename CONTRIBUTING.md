# Contributing to Stego.Image

Thank you for your interest in contributing to Stego.Image! This project is a client-side React/Vite app for hiding and extracting files inside images using encryption and compression.

## ✅ Ways to contribute

- Report bugs or suggest features by opening an issue
- Submit improvements or bug fixes via pull request
- Improve documentation and examples
- Fix typos, formatting, or accessibility issues

## 🚀 Getting started

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/<your-username>/StegoImage.git
   cd StegoImage
   ```
3. Install dependencies:
   ```bash
   npm install
   ```
4. Start development server:
   ```bash
   npm run dev
   ```

## 🧰 Branching and workflow

- Create a new branch for every change:
  - `feature/<short-description>` for new features
  - `fix/<short-description>` for bug fixes
  - `docs/<short-description>` for documentation changes
- Keep your branches focused and small
- Rebase or merge from `main` before opening a pull request

## 🔧 Development commands

- `npm run dev` — run the app locally
- `npm run build` — create a production build
- `npm run lint` — run ESLint across the repository
- `npm run preview` — preview the production build locally

## 🧹 Code style

- Follow existing project conventions for React, JSX, and CSS
- Use `camelCase` for variables and functions
- Keep components small and composable
- Prefer clear variable names and descriptive comments
- Ensure new code is easy to read and maintain

## 🧪 Testing your changes

This repository does not currently include automated test coverage. Please manually verify that:

- UI behavior works in the browser
- Encode and decode flows behave correctly
- New features do not break existing functionality
- Lint passes with `npm run lint`

## 📝 Pull request guidelines

- Provide a clear title and description for your PR
- Explain the problem and how your change fixes it
- Reference related issues when applicable
- Keep changes scoped to a single purpose
- Make sure the app builds successfully

## 💬 Reporting issues

When opening an issue, include:

- A descriptive title
- Steps to reproduce the problem
- Expected versus actual behavior
- Browser and environment details if relevant

## 📦 Documentation improvements

Contributions to documentation are welcome. If you see opportunities to improve the README, examples, or wording, please submit a PR.
